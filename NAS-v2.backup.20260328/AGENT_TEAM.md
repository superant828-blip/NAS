## Agent 职责定义

### 1. 项目管理 Agent (PM)
**职责**: 任务规划、进度跟踪、多Agent协调

**具体任务**:
- 制定迭代计划
- 分配任务给其他Agent
- 监控进度
- 汇总结果

**使用场景**: 启动项目、多Agent任务

---

### 2. 后端开发 Agent (Backend)
**职责**: API开发、数据库、业务逻辑

**子方向**:
- **API开发** - FastAPI接口
- **存储管理** - ZFS/文件操作
- **安全认证** - JWT/用户权限

**使用场景**: 新功能开发、API修复

---

### 3. 前端开发 Agent (Frontend)  
**职责**: 用户界面、交互体验

**子方向**:
- **Vue开发** - 组件开发
- **UI/UX** - 界面优化
- **页面交互** - 动效、反馈

**使用场景**: 页面开发、UI优化

---

### 4. 运维 DevOps Agent
**职责**: 部署、监控、日志

**子方向**:
- **部署** - Docker/脚本
- **监控** - 系统状态
- **日志** - 错误追踪

**使用场景**: 部署上线、故障排查

## 任务分配示例

### 功能开发任务
```
任务: 添加文件分享功能
分配:
├── Backend Agent → API: /api/v1/share
├── Backend Agent → 数据库: shares表
├── Frontend Agent → 分享页面UI
└── DevOps Agent → 测试部署
```

### Bug修复任务
```
任务: 登录页面空白
分配:
├── Frontend Agent → 检查前端代码
└── Backend Agent → 检查API响应
```

## Agent 通信协议

### 任务格式
```
## 任务：[任务名称]

### 背景
[简要说明]

### 具体要求
1. [要求1]
2. [要求2]

### 交付物
- [文件1]
- [文件2]
```

### 结果格式
```
## 完成报告

### 修改文件
| 文件 | 说明 |
|------|------|
| xxx.py | 修改了xxx功能 |

### 测试结果
| 功能 | 状态 |
|------|------|
| xxx | ✅ 通过 |
```

## 常用 Agent 命令

### 启动后端开发
```
启动 agent-api-dev: 新功能开发
启动 agent-storage-dev: 存储功能开发
启动 agent-auth-dev: 认证功能开发
```

### 启动前端开发
```
启动 agent-ui-dev: 界面开发
启动 agent-interaction: 交互优化
```

### 启动运维
```
启动 agent-deploy: 部署配置
启动 agent-monitor: 监控设置
```
